Preface

This document describes wxCL = wxWidgets + Common Lisp; common lisp binding to wxWidgets library.

Bugs both in this document and in wxCL itself should be reported to the mailing list < wxcl-devel/at/lists.sourceforge.net >.

Downloading and Installing wxCL

Window Users: Download and unzip the library. The wxCL project provides a "*.dll" file in the miscellaneous folder. The file wxcl.lisp in the folder "wxcl/src" defines the constant variable +library-name+ which contains the relative path to the aforementioned dll file.

Linux/Mac Users: To use the library you need to compile and make a dynamically linked library for your platform. The instructions for building and linking the library can be found at wxHaskell project: http://wxhaskell.sourceforge.net/building.html

Before compiling and loading the wxCL library, edit the clisp-wrappers/wxCL.lisp, and make sure that the variable +library-name+ is pointing to the correct location. To load the library you can use asdf or asdf-install. The wxcl.asd file makes sure that all the files are loaded in the correct order. If you want you can also create a memory image and save it for faster loading.
Dr. Edmund Weitz has written a nice tutorial on how to use asdf-install.

Running Examples

To run the examples which are part of wxCL, you need to first go to the folder containing the examples file, and then load the example file. All the samples assume that wxCL is installed and they will try to load it using asdf.

Programming with wxCL

This release of wxCL is not backward compatible with the earlier releases. The src folder contains all the CLOS source code and the ffi wrapper code. Note:When the program ends, the destructor for the wrappers calls the callback functions with a null event, so that they can do any cleaning up (if necessary). So, its always a wise thing to make sure that in your call back functions, there is an `if` condition checking whether the event is null or not.

Hope this helps, more information can be found by looking at the examples, if you still have doubts then ask on the mailing list.
