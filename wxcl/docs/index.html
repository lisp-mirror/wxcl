Preface

This document describes wxCL = wxWidgets + Common Lisp; a common lisp GUI library with binding to wxWidgets.

Bugs both in this document and in wxCL itself should be reported to the mailing list < wxcl-devel/at/lists.sourceforge.net >.

Downloading and Installing wxCL

Window Users: Download and unzip the library. The wxCL project provides a "*.dll" file in the lib folder. The file `defpackage.lisp' in the folder "wxcl/src" defines the constant variable +library-name+ which contains the relative path to the aforementioned dll file.

Linux/Mac Users: To use the library you need to compile and make a dynamically linked library for your platform. The instructions for building and linking the library can be found at wxHaskell project: http://wxhaskell.sourceforge.net/building.html. Use the `wxC' code from the wxCL cvs repository to build the library.

Before compiling and loading the wxCL library, edit the `wxcl/src/defpackage.lisp', and make sure that the variable +library-name+ is pointing to the correct location. To load the library you need to have asdf. The wxcl.asd file makes sure that all the files are loaded in the correct order. If you want you can also create a memory image and save it for faster loading.
Dr. Edmund Weitz has written a nice tutorial on how to use asdf-install.

Running Examples

To run the examples which are part of wxCL, you need to first go to the folder containing the examples file, and then load the example file. All the samples assume that wxCL is installed and they will try to load it using asdf.

Programming with wxCL

This release of wxCL is not backward compatible with the earlier releases. The src folder contains all the CLOS source code and the ffi wrapper code. Have a look at the examples, and it should not be too hard to get started with wxCL.

Hope this helps, and that you have fun time using the library. If you have any doubts, suggestions, if you want to help us, etc then send an email to the mailing list.
