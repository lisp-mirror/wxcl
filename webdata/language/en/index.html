Welcome!

The wxCL Project Group works on the development of an
  • industrial strength interface
  • under the wxWidgets (formerly wxWindows) Open Source License
  • for the wxWidgets windowing system
  • to various implementations of the Common Lisp programming language
  • on various operating-system platforms.
We'd appreciate if you take some time to tell us why you favor the blend of wxWidgets with Common Lisp over other windowing systems and which aspects are important for you in this decision. Your hints help us in the development of the interface.

Announcement!

We are pleased to announce the Release 1.3.0 Alpha of wxCL, the latest installment in wxCL development effort. In this release wxCL now uses CFFI as a backend.

The current release of wxCL 1.3.0 should run on all the common lisp implementations supported by CFFI. It is tested only on Windows XP and Mac OS X but it should work on Linux as well (not tested due to lack of resources).

If you face any problem on any platform or common lisp implementation, feel free to contact us on the mailing list, we will try to help you.

If you are a Common Lisp hacker, we will appreciate if you join our development team and help us in improving and designing the library. Contributors (including newbies) with all levels of expertise are welcome. Even if you cannot contribute with code, helping us in making design choices, reporting bugs, etc.

The best place to ask questions, and/or give feedback on the library is the wxCL-devel mailing list.
