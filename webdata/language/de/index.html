Willkommen!

Die wxCL-Projektgruppe arbeitet an der Entwicklung einer
  • industriegerechten Schnittstelle
  • für das wxWidgets Windowing-System,
  • welches unter der wxWidgets (früher wxWindows) Open Source Lizenz steht,
  • für verschiedene Implementierungen der Common Lisp-Programmiersprache,
  • lauffähig auf verschiedenen Betriebssystem-Plattformen.
Es wäre uns eine große Hilfe, wenn Sie uns schreiben könnten, warum Sie wxWidgets mit Common Lisp gegenüber anderen Windowing-Systemen vorziehen und welche Aspekte wichtig für diese Entscheidung sind. Ihre Hinweise helfen uns beim Entwickeln der Schnittstelle.

Bekanntmachung!

Es ist uns eine Freude, den neuen Release 1.3.0 Alpha von wxCL, das aktuelle Resultat des wxCL-Entwicklungsaufwandes, bekanntzugeben. In diesem Release verwendet wxCL nun CFFI als Backend.

Der aktuelle Release von wxCL 1.3.0 sollte mit allen Common Lisp-Implementationen funktionieren, die CFFI unterstützen. Er wurde unter Windows XP und Mac OS X getestet und sollte auch unter Linux funktionieren (aufgrund mangelnder Ressourcen nicht getestet).

Wenn Sie auf irgendein Problem stoßen, kontaktieren Sie uns über die Mailing-Liste - wir werden versuchen, Ihnen zu helfen.

Wenn Sie ein Common Lisp-Hacker sind, möchten wir sie freundlich dazu auffordern, dem Team beizutreten und uns dabei zu helfen, die Bibliothek zu gestalten und zu verbessern. Helfer mit den verschiedensten Erfahrungsstufen (auch Neulinge) sind uns willkommen. Auch wenn Sie keinen Code schreiben können, helfen Sie uns, indem Sie Designentscheidungen treffen, Fehler melden, usw.

Der beste Ort, Fragen zu stellen und/oder Feedback zur Bibliothek abzugeben, ist die wxCL-devel Mailing-Liste.
