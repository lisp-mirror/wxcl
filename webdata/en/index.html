The wxCL Project Group

works on the development of an

  • industrial strength interface
  • under the wxWidgets (formerly wxWindows) Open Source License
  • for the wxWidgets windowing system
  • to various implementations of the
    Common Lisp programming language
  • on various operating-system platforms.
  • To know more about the wxWidgets Open Source License, visit this page of the wxWidgets project group.
  • We'd appreciate if you take some time to tell us in an eMail why you favor the blend of wxWidgets with Common Lisp over other windowing systems and which aspects are important for you in this decision. Your hints help us in the development of the interface.

wxCL 1.1.1 Alpha

Sep. 23, 2005:

We are pleased to announce Release 1.1.1 Alpha of wxCL, the latest installment in wxCL development effort. This release now includes the wxWidgets 2.6.2 Dynamic Link Library. It introduces a number of bug fixes and some API enhancements. This version is more stable with native CLISP on MS Windows XP.

Please see the Change Log for a complete list of changes, and read the Quick Start Guide before using this version as using the wxWidgets 2.6.2 DLL requires you to close the library every time you finish the program. Also, don't forget to take a look at the CVS version of wxGlade, which provides a cool way of creating wxCL GUIs.

wxCL aims to provide an industrial strength GUI library to application programmers which is portable across Common Lisp implementations and operating system platforms. It builds upon wxWidgets, a comprehensive C++ library that provides uniform application interface to all major GUI platforms; including GTK+, MS Windows, X Windows, and Apple MacOS X.

  • The current Release 1.1.1 of wxCL runs on CLISP, and is tested on MS Windows XP but should work on Linux and Apple MacOS X as well, although not tested due to lack of resources. wxCL can be downloaded from
    http://www.wxcl-project.org/
    en/downloads--downloads-1.html
  • The best place to ask questions, report bugs and / or give feedback on the library is the wxCL-devel mailing list; see details at
    http://www.wxcl-project.org/
    en/contact--contact-1.html
  • We need your help: If you are a Common Lisp, CPP, or C hacker, please send us an eMail.

Other Resources & Related Projects

  • Recently, a discussion on the CLISP-devel mailing list gave the work on wxCL a new spin; see http://article.gmane.org/gmane.lisp.clisp.general/9757.
  • SWIG_CLISP is a new module in SWIG, producing wrapper code for the C interface of CLISP. Its maintainer is a member of the wxCL-devel mailing list.
  • The project groups wxHaskell, wxOcaml, and wxEiffel use similar C libraries to bind the member functions in the CPP class hierarchy of wxWidgets to simple C function calls. A project named wxC aims to unify these slightly differing libraries.
