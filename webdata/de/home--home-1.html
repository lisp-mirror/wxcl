Die wxCL Projektgruppe

arbeitet an der Entwicklung einer

  • industriellen Anforderungen
    entsprechenden Schnittstelle
  • unter der wxWidgets (früher wxWindows) Open-Source-Lizenz
  • für das Fenstersystem wxWidgets
  • zu verschiedenen Implementationen der
    Programmiersprache Common Lisp
  • auf verschiedenen Betriebssystem-Plattformen.
  • Besuchen Sie diese Seite der wxWidgets Projektgruppe, um mehr über die wxWidgets Open-Source-Lizenz zu erfahren.
  • Wir würden uns freuen, wenn Sie sich etwas Zeit nehmen, um uns in einer eMail zu sagen, warum Sie die Verbindung von wxWidgets mit Common Lisp anderen Fenstersystem vorziehen und welche Aspekte Ihnen bei dieser Entscheidung wichtig sind. Ihre Hinweise helfen uns bei der Entwicklung der Schnittstelle.

wxCL 1.1.1 Alpha

23. Sep. 2005:

We are pleased to announce Release 1.1.1 Alpha of wxCL, the latest installment in wxCL development effort. This release now includes the wxWidgets 2.6.2 Dynamic Link Library. It introduces a number of bug fixes and some API enhancements. This version is more stable with native CLISP on MS Windows XP.

Please see the Change Log for a complete list of changes, and read the Quick Start Guide before using this version as using the wxWidgets 2.6.2 DLL requires you to close the library every time you finish the program. Also, don't forget to take a look at the CVS version of wxGlade, which provides a cool way of creating wxCL GUIs.

wxCL aims to provide an industrial strength GUI library to application programmers which is portable across Common Lisp implementations and operating system platforms. It builds upon wxWidgets, a comprehensive C++ library that provides uniform application interface to all major GUI platforms; including GTK+, MS Windows, X Windows, and Apple MacOS X.

  • The current Release 1.1.1 of wxCL runs on CLISP, and is tested on MS Windows XP but should work on Linux and Apple MacOS X as well, although not tested due to lack of resources. wxCL can be downloaded from
    http://www.wxcl-project.org/
    de/downloads--downloads-1.html
  • The best place to ask questions, report bugs and / or give feedback on the library is the wxCL-devel mailing list; see details at
    http://www.wxcl-project.org/
    de/contact--contact-1.html
  • We need your help: If you are a Common Lisp, CPP, or C hacker, please send us an eMail.

Weitere Resourcen & verbundene Projekte

  • Durch eine Diskussion auf der CLISP-Mailing-Liste hat die Arbeit an wxCL vor kurzem einen neuen Anstoß erhalten; vgl. http://article.gmane.org/gmane.lisp.clisp.general/9757.
  • SWIG_CLISP ist ein neues Modul in SWIG, dass Umhüllungs-Kode für die C-Schnittstelle von CLISP erzeugt. Sein Entwickler ist Mitglied der wxCL-devel-Mailing-Liste.
  • Die Projektgruppen wxHaskell, wxOcaml und wxEiffel nutzen ähnliche C-Bibliotheken, die die Elementfunktionen in der CPP-Klassenhierarchie von wxWidgets an einfache C-Funktionsaufrufe binden. Ein Projekt namens wxC bemüht sich um die Vereinheitlichung dieser leicht unterschiedlichen Bibliotheken.
